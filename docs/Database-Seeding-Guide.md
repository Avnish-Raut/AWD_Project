# Database Seeding Guide
**Smart Event Management Platform**

This guide explains how the mock data generation works and provides the exact steps required to populate a fresh database in a new environment (like the evaluation VM).

---

## 1. What is Database Seeding?

Database seeding is the process of populating a blank database with initial "mock" data. Instead of manually registering users and creating events one at a time for testing or grading, the grading environment can execute a single script to generate a fully populated, realistic application state.

The seed logic is located at: `backend/prisma/seed.ts`

### What Data Gets Injected?
When the seed script is run, it performs the following automatic operations:
1. **Data Sanitization:** First, it securely wipes the entire database clean to prevent duplicate data errors on consecutive runs. 
2. **User Creation:** It generates 8 core users, assigning them specific system roles using `bcrypt` to strictly hash their passwords.
   - **Admin (1):** Super user assigned the `ADMIN` role.
   - **Organizers (2):** Event managers assigned the `ORG` role.
   - **Students (5):** Regular users assigned the `USER` role.
3. **Event Generation:** It generates 4 distinct application events (from hackathons to cloud workshops) with calculated dates relative to the time the script is executed. 
4. **Registrations:** It builds cross-table relationships by automatically registering combinations of those 5 generated students into the newly generated events.
5. **Logs:** It generates initial system activity logs.

## 3. How to Import the Seed to a Fresh Database

If you are setting up the project on a new VM, follow these steps to deploy the PostgreSQL schema and populate the test data.

### Prerequisites
- Ensure PostgreSQL is installed and running matching the `DB-config.md` instructions.
- Ensure the `backend/.env` file is properly configured with your specific `DATABASE_URL`.
- Ensure you have installed the backend packages using `npm install`.

### Execution Steps
Open your terminal and navigate to the backend folder:

```bash
cd backend
```

**Step 1: Push the Schema**
This command reads the `schema.prisma` file and creates the raw SQL structure (tables, columns, foreign keys) in the PostgreSQL database.
```bash
npx prisma db push
```

**Step 2: Generate the Client**
This command binds the newly created PostgreSQL schema to the native TypeScript application code used by the Node.js backend.
```bash
npx prisma generate
```

**Step 3: Run the Seed**
This command executes the `seed.ts` script. It will connect to your PostgreSQL database using the `DATABASE_URL` and begin injecting all users, events, and registrations simultaneously.
```bash
npx prisma db seed
```

### Verification
If successful, the terminal will print:
```text
🌱 Starting database seeding...
Cleaning existing data...
Creating users (Admins, Organizers, Participants)...
✅ Database seeding complete!
🌱 The seed command has been executed.
```

Your VM now has a fully functional, populated Smart Events database ready for evaluation!
